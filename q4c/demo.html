Resource-based Health Information Zone for the Mobile Enterprise

Rhizome is a set of directives and servicesI am building using AngularJS and other JavaScript libraries. The idea is to create a set of client-side components that will support standards-based mobile and social application development for the Healthcare industry.

Rhizome: Questions4Care, shown here, demonstrates how a Questionnaire application can use both machine-readable elements (codes and code systems) and human-readable elements, like PDF and Scalable Vector Graphics (SVG).

Rhizome makes ample use of an emerging Health Information standard called HL7 Fast Healthcare Information Resources (FHIR), which support interoperability through a resource-based approach to JSON- and XML-messaging. FHIR is quite a recent standard, but it is gaining traction as a way to work with Resource-based Health Information.

Interoperability often relies on a combined notion of human- and machine-readable content. HL7 FHIR resources usually use HTML5 for the human-readable portion, but you could also use an image like a JPEG or PNG, and in this case, I have used SVG, which allows me to use a JavaScript library called SnapSVG to animate the questionnaire whenever coded answers are selected, using a "frame-and-easing" algorithm.
